# СтудЧР-2026 ЭК

Источник структуры: открытая вкладка Google Sheets через CDP и локальный генератор `~/google_sheet_writer/examples/studchr/generate_ek.py`.

## Листы-референсы

- `Сетка` — общая сетка боев, команды, суммы, места.
- `Площадки` — номер, место, ведущий, ассистент.
- `1/16`, `1/8`, `1/4`, `1/2`, `Финал` — протоколы боев.
- `Пересев перед 1/4` — ранжирование 12 команд после `1/8`.
- `Статистика`, `_stats_team`, `_stats_ind`, `Табло`, `Составы`, `Жеребьёвка` — вспомогательные листы.

## Этапы и бои

- `1/16, заход 1`: 6 боев `A-F`, по 4 команды.
- `1/16, заход 2`: 6 боев `G-L`, по 4 команды.
- `1/8`: 6 боев `M-R`, по 4 команды.
- `Пересев перед 1/4`: неигровой этап `stage_type=reseed`.
- `1/4`: 4 боя `S-V`, по 3 команды.
- `1/2`: 2 боя `W-X`, по 4 команды.
- `Финал`: 1 бой `Y`, 4 команды.

## JSON-артефакт

Сетка генерируется командой:

```bash
python3 scripts/generate_studchr_grid.py > /tmp/studchr-ek-2026.json
```

Полученный JSON вставляется на `/host/tournament/{id}/import` и пересоздает игру выбранного турнира в SQLite. UI сетки не должен зависеть от hardcoded СтудЧР: он строит HTML по импортированной JSON-схеме.

## Переходы

- `1/16`: старт из 4 корзин жеребьевки, позиции 1-6 идут в `A-F`, позиции 7-12 — в `G-L`.
- `1/8`: пары соседних боев первого этапа.
  - `M`: `A1, G1, B2, H2`; `N`: `B1, H1, A2, G2`.
  - `O`: `C1, I1, D2, J2`; `P`: `D1, J1, C2, I2`.
  - `Q`: `E1, K1, F2, L2`; `R`: `F1, L1, E2, K2`.
- Пересев после `1/8`: участники — места 1-2 из `M-R`, сортировка по сумме мест, сумме, `Σ+`, 50, 40, 30, 20, жребию.
- `1/4`: `S=[1,8,9]`, `T=[4,5,12]`, `U=[2,7,10]`, `V=[3,6,11]` по номерам пересева.
- `1/2`: `W=[S1,T2,U1,V2]`, `X=[S2,T1,U2,V1]`.
- `Финал`: `Y=[W1,W2,X1,X2]`.

## Площадки

В живой таблице видны площадки `1-6`: `Москва-1` ... `Москва-5`, `Рим`, плюс отдельный воскресный блок. Для приложения базовая сущность — только номер и название; ведущий/ассистент можно добавить позже без ломки модели.
