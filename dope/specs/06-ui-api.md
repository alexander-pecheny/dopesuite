# UI и API

## Интерфейс

- Не копируем Google Sheets. Используем текущий язык проекта: `host-top`, `sheet-frame`, `match-table`, sticky-колонки, существующие цвета и состояния.
- Основной экран `/host` и `/viewer` — вся турнирная сетка на одной странице: один этап = одна вертикальная колонка; внутри колонки бои этапа идут сверху вниз.
- Сетка генерируется из импортированного JSON и текущего состояния SQLite.
- Для каждого этапа на странице видны все бои этапа, как в рабочей Google-таблице, без перехода в отдельный бой как основной сценарий.
- Заходы одного спортивного раунда можно хранить как разные этапы, если так удобнее для ведения. Для СтудЧР `1/16` разбита на `заход 1` (`A-F`) и `заход 2` (`G-L`).
- Отдельный экран боя допустим только как техническое/финальное исключение; обычное ведение этапа должно происходить с общей страницы этапа/сетки.
- Страница площадок показывает таблицу `№ / Название`; в host-режиме название редактируется inline.
- Страница `/import` содержит вставку JSON-схемы и создает турнир на лету.

## Роуты страниц

- `/import` — вставка JSON и импорт турнира.
- `/host` — вся сетка чемпионата для ведущего.
- `/host/venues` — редактирование площадок.
- `/viewer` — вся зрительская сетка.
- `/viewer/venues` — read-only площадки.
- `/host/matches/Y`, `/viewer/matches/Y` — опциональный отдельный экран финала.

## API

- `POST /api/import` — принять JSON-схему и пересоздать текущий турнир.
- `GET /api/tournament` — схема, этапы, бои, площадки, текущие результаты.
- `GET /api/matches/{code}` — полное состояние одного боя.
- `POST /api/matches/{code}/update` — отметки ответов, игроки, места, перестрелка.
- `POST /api/matches/{code}/finish` — закончить или открыть бой.
- `POST /api/matches/{code}/venue` — заменить площадку боя.
- `GET /api/venues`, `PUT /api/venues/{number}` — список и редактирование площадок.
- `GET /events` — SSE, как сейчас, но события имеют `scope`: `tournament`, `match:{code}`, `venues`.

## Совместимость

Логику подсчета одного боя переносим почти без изменений. Старые `/api/state` и `/api/update` можно временно оставить как алиасы для активного боя на время миграции.
