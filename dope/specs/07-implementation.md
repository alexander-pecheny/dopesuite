# План внедрения

## Шаги

1. Добавить слой SQLite: подключение, миграции, репозитории, тестовую БД. Готово.
2. Сгенерировать референсный JSON СтудЧР отдельной командой. Готово: `cmd/generate-studchr-grid`.
3. Добавить `/import`: вставка JSON и создание турнира на лету. Готово.
4. Добавить общую сетку турнира на `/host` и `/viewer`, построенную из JSON и SQLite. Готово для структуры турнира.
5. Вынести расчет боя из `MatchState` в функции, работающие с DB-моделью. Частично готово для старого экрана боя.
6. Реализовать ведение всех боев этапа на одной странице общей сетки.
7. Реализовать разрешение слотов `seed`, `fromMatch`, `reseed`.
8. Добавить пересев после этапа и регрессионные тесты для схемы СтудЧР.

## Тесты

- Unit: счет темы, счет команды, перестрелка, ручные места.
- DB: миграции, импорт схемы, ограничение состава до 9 игроков.
- Tournament: импорт JSON СтудЧР, переходы `A-L -> M-R`, пересев `M-R -> S-V`, полуфиналы, финал.
- API: редактирование боя, завершение, смена площадки, SSE revision.

## Миграция текущего состояния

Текущий `match_state.json` можно импортировать как один бой `A` в тестовый турнир. Это не основной формат хранения, а одноразовый мост для локальной разработки.
