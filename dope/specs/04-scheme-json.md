# JSON-схема турнира

JSON описывает спортивную схему, а SQLite хранит конкретное состояние турнира.
JSON не зашивается в код сервера: его можно сгенерировать отдельной командой,
вставить на странице `/import`, и из него будет создан текущий турнир.

См. также: [04a-scheme-sources.md](04a-scheme-sources.md).

## Верхний уровень

```json
{
  "schemaVersion": 1,
  "slug": "studchr-ek-2026",
  "title": "СтудЧР-2026, ЭК",
  "questionValues": [10, 20, 30, 40, 50],
  "regularThemeCount": 12,
  "venues": [{"number": 1, "title": "Москва-1"}],
  "stages": []
}
```

Референсный JSON СтудЧР генерируется отдельно:

```bash
go run ./cmd/generate-studchr-grid > static/schemes/studchr-ek-2026.json
```

Пересев задается в `stages` как отдельный этап с `stage_type: "reseed"`. Это сохраняет линейную последовательность турнира: игровой этап, пересев, следующий игровой этап.

## Этап и бой

```json
{
  "code": "r8",
  "title": "1/8 финала",
  "stage_type": "matches",
  "position": 2,
  "matches": [{
    "code": "M",
    "title": "Бой M",
    "venue": 1,
    "participantCount": 4,
    "slots": [
      {"fromMatch": {"match": "A", "place": 1}},
      {"fromMatch": {"match": "G", "place": 1}},
      {"fromMatch": {"match": "B", "place": 2}},
      {"fromMatch": {"match": "H", "place": 2}}
    ]
  }]
}
```

## Договоренности

- `stage_type="matches"` содержит `matches`.
- `stage_type="reseed"` содержит настройки пересева вместо боев.
- `venue` — номер дефолтной площадки, не `venue_id`.
- `participantCount` задается явно, чтобы поддерживать бои на 3 и 4 команды.
- `slots` идут в порядке отображения команд в протоколе.
- Опциональный `layout` хранит подсказки отображения. В текущем UI один этап показывается одной колонкой, поэтому заходы вроде `1/16 A-F` и `1/16 G-L` лучше задавать отдельными stage.
- В конкретном турнире ведущий может заменить площадку у боя без изменения JSON.

## Импорт

- `/import` принимает JSON целиком и отправляет его в `POST /api/import`.
- Импорт пересоздает текущий турнир в SQLite: схемы, этапы, бои, слоты и площадки.
- Слоты с `seed`, `fromMatch` и `reseed` импортируются как источники без команды; на сетке они отображаются как `К1-1`, `A1`, `Пересев-8` и т.п.
- Слоты с `team` могут сразу создать известную команду, если в объекте есть `name` или `label`.
