Dope refactor pass

Branch vibe-refactor · 2026-05-24 · backend + frontend cleanup, no behavior change intended.

What I did

  • Centralised the floating-name popover. The mouse/focus/scroll plumbing for tooltip popovers existed twice (~110 lines each in host.js and viewer.js). Extracted into gameTable.createFloatingPopover({root, specs}) in match-table.js; each page now just passes its own spec list and binds.
  • Pulled shared helpers into match-table.js: normalizeVenue, formatVenue, formatBattleVenue, formatBattleVenueShort, statusLabel, formatNumber, formatPlace, stageType, stageTabLabel, teamListCell, buildVenuesTable (with {editable, onTitleChange} option). Host and viewer destructure them off window.DopeTable.
  • Deleted dead code. buildFestTable and buildSubnav were defined in both host.js and viewer.js but never called — replaced by fest-grid.js's buildFestGrid a while back. Gone now. Also removed local clampNumber / clamp / cssEscape wrappers and other thin pass-throughs.

Bugs fixed along the way

  • Viewer venue label inconsistency. Viewer's formatBattleVenue was пл. 1: Москва-1; host and grid used пл. 1 (Москва-1). Same data, two formats — looked like a stale copy. Now both use parens.
  • JSON-patch DoS via array index. parseJSONPatchPath in scoped_api.go accepted any non-negative integer; applyJSONSet then grew the slice up to that index. An authenticated editor could allocate gigabytes by patching [1_000_000_000, …]. Capped to maxPatchArrayIndex = 4096.
  • cdp.py device desktop was broken. CDP's setTouchEmulationEnabled rejects maxTouchPoints: 0 even when enabled: false. Switching to the desktop profile crashed with "Touch points must be between 1 and 16". Now we always send ≥ 1.
