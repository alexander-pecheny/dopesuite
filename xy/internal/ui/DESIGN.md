# ui — constrained typed UI DSL

Every page in xy is authored in `.xui` (a line-oriented DSL) or via the typed Go
builder, both producing the same node tree, rendered to HTML by one deterministic
printer, validated against one closed vocabulary (`vocab.json`). Off-system usage
(unknown element/attribute/class, duplicate id) is a compile error. No HTML text
exists in app code. See `constrained-ui-brief.md` (repo root) for the rationale.

## Package layout

```
internal/ui/
  vocab.json    closed vocabulary: elements + per-element attrs, global attrs,
                attr patterns (aria-*, data-*), void set, class-token whitelist
  vocab.go      //go:embed vocab.json, load into lookup tables
  node.go       node tree types
  render.go     printer (node tree → []byte HTML)
  validate.go   validator (tree vs vocab; duplicate-id check)
  parse.go      .xui parser (source → tree), errors carry file:line
  compile.go    Compile(name string, src []byte) ([]byte, error) = parse+validate+render
  builder.go    hand-written builder core (Item/Attr/Node, Text, Line, Comment, Blank, Block/Inline)
  tags_gen.go   GENERATED by cmd/uigen from vocab.json: element ctors (Div, P, …),
                attr ctors (Href, Type, …), ClassToken consts (AuthHint = "auth-hint", …)
cmd/uigen/      codegen: vocab.json → tags_gen.go (wired as //go:generate in vocab.go)
cmd/uic/        CLI: `go run ./cmd/uic file.xui` → HTML on stdout, errors on stderr
web/assets/ui/  the app's pages, *.xui (embedded alongside static/)
```

## .xui grammar

Line-oriented. Indentation = tree depth, exactly 2 spaces per level. Tabs forbidden.

Per line (after indent):
- `# …` — source comment, ignored (never rendered).
- `-- text to end of line` — HTML comment. Consecutive `--` lines at the same
  indent merge into ONE multi-line comment node.
- `doctype` — emits `<!doctype html>`. Only valid as the first node.
- blank line — emits a blank output line (preserved verbatim, between siblings).
- element line: `tag attr* inline-item*`
  - `tag`, attr names: `[a-z][a-z0-9-]*`.
  - attr: `name="value"` (always double-quoted) or bare `name` (boolean).
    Order is preserved exactly — `class` is a normal attribute, no shorthand.
  - inline-item: `"string"` (escapes: `\"`, `\\`, `\n`) or `(tag attr* inline-item*)`
    — a nested inline element, parens nest arbitrarily.
  - An element line with inline-items is an inline element (content on one output
    line). An element line followed by deeper-indented lines is a block element.
    Having both is a parse error. Neither → empty element.
- inline-run line: a child line starting with `"` or `(` — a sequence of
  inline-items rendered as ONE output line (text and elements concatenated with
  no separator). Example: `"🔔" (span class="notif-badge" id="notifBadge" hidden)`.

## Printer rules (these reproduce the repo's existing HTML style exactly)

- Output indent = 2 spaces × depth, EXCEPT: `<html>`'s children render at the
  same depth as `<html>` (head/body sit at column 0).
- `doctype` → `<!doctype html>`.
- Void elements (html5 set: meta, link, input, br, img, hr, source, …) → `<tag …>`,
  no closing tag, no self-closing slash.
- Block element → `<tag …>` line, children each on own lines, `</tag>` line.
- Inline element → `<tag …>content</tag>` on one line. Empty non-void → `<tag …></tag>`.
- Inline-run child line → one output line at child indent.
- Comment, single line → `<indent><!-- text -->`. Multi-line → first line
  `<indent><!-- line1`, continuations at indent+5 spaces (aligns under text),
  last continuation ends with ` -->`.
- Attributes: `<tag name="v" bare name2="v2">` — authored order, single spaces.
- Escaping: text nodes escape `&` `<` `>` only (NOT quotes — pages contain « » … → 
  literally). Attr values escape `&` `<` `>` `"` and newline → `&#10;`.
- Every rendered page ends with a trailing newline.

## Validation (compile errors, with file:line from the parser)

- element not in vocab; attribute not allowed for that element
  (allowed = global attrs ∪ element's attrs ∪ patterns `aria-*`, `data-*`);
- `class` value: split on spaces, every token must be in the class whitelist;
- duplicate `id` value within one page;
- indentation not a multiple of 2 / jumping more than one level;
- inline-item element must itself be inline (no block children — guaranteed by
  grammar in .xui, checked for builder-made trees).

## Go builder (the DSL's programmatic form — used by dynamic pages, e.g. /admin)

```go
ui.Div(ui.Class(ui.AppearanceModal), ui.ID("x"), ui.Aria("modal", "true"), children…)
```
- `Item` interface implemented by `Attr` and `Node`; element ctors are
  `func Div(items …Item) *Element` (generated). Attr ctors generated per vocab
  (`Href(v)`, `Type(v)`, bare: `Hidden()`, `Required()`…); `Aria(name, v)`,
  `Data(name, v)` (empty v → bare), `Class(tokens …ClassToken)`.
- `Text(s)`, `Line(items…)` (inline-run), `CommentNode(lines…)`, `Blank()`.
- Content mode: children that are all Text → inline automatically; any element
  child → block; `ui.Inline(…)` / `ui.Line(…)` force one-line content explicitly.
- `Render(doc)` validates then prints; validation failures return errors (a page
  is also exercised by tests, so misuse fails in CI, not just at runtime).

## Server integration

- `web/assets/assets.go`: `//go:embed all:static all:ui`.
- `servePage("ui/board.xui")`: embed mode compiles every page once at startup
  (compile error = panic at boot); disk (dev) mode recompiles per request
  (error → 500 with the compile message). Everything downstream — versionAssetRefs,
  CSP headers, gzip — is unchanged and operates on the compiled bytes.
- `/admin` pages render builder trees per request (same headers as before).

## Parity contract (conversion-time verification)

- The 7 static pages: **byte-identical** to their pre-conversion HTML
  (`git show <pre-conversion>:web/assets/static/X.html` vs `go run ./cmd/uic web/assets/ui/X.xui`).
- Admin pages (dynamic): **DOM-identical** for representative data; whitespace
  between block tags may differ (the old html/template control-flow lines left
  indentation artifacts that are not reproduced).
