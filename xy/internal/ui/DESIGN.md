# xy/internal/ui — the xy overlay on DopeUIKit's kit

The constrained UI DSL now lives in **DopeUIKit** (vendored via
`replace => ../dopeuikit`), split into the generic engine (`pecheny.me/
dopeuikit/ui`) and the shared design system (`pecheny.me/dopeuikit/kit`). Its
`DESIGN.md` is the authoritative spec (grammar, validation, extension API, the
core primitive catalog). This package is xy's thin **overlay on the kit**: it
builds the xy `App` via `kit.NewApp` and re-exports the kit's builder so xy code
imports one package.

## What lives here

- `vocab.json` — the xy overlay vocabulary: new primitives (`docoverlay`,
  `headrow`, `headactions`, `split`, `pane`, `previewtitle`), new enums
  (`pane-kind`, `doc-kind`, `doc-variant`, `head-kind`) and values added to core
  enums (`page-kind` +wide/board, `checkbox-kind` +preview/card-preview,
  `editor-kind` +comment/handouts/importsrc, `mount-kind` = the board kinds).
- `expand.go` — the xy expanders: `checkbox`/`editor` **override** the core
  generic ones (xy's `attach-lossless`/`card-desc` flavors), the five overlay
  primitives, and `xyMounts` (mount kind → tag+classes).
- `app.go` — constructs the xy `App` via `kit.NewApp` with the overlay vocab,
  expanders, mounts, and xy's PWA `Chrome` (no-zoom viewport, `menu.js` boot,
  page kinds sheet/full/wide/board). Exposes `Compile` (`.dopeui` → HTML) and
  `Render` (builder tree → HTML, used by the dynamic `/admin` pages).
- `tags_gen.go` — generated by `go generate` (`cmd/uigen -overlay vocab.json
  -base .../kit`): overlay constructors/enum consts plus re-export shims for every
  kit constructor/const, so `xy/internal/ui` is the single import.

Pages are authored in `web/assets/ui/*.dopeui` and compiled at serve time
(`servePage`). CSS: the kit serves the shared design system as `core.css`; xy's
`web/assets/static/styles.css` is now just the xy layer (kanban/card/board +
xy vars + PWA overrides), concatenated after core at serve time.
